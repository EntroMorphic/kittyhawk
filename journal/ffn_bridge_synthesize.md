# Ternary FFN bridge — SYNTHESIZE

Executable specification.

---

## Architecture

```
pixels → normalize → direct quantize → LSH
                                         │
                                    ┌────┤
                                    │    │
                               LSH pred  routing vector
                                    │    (M labels + M distances)
                                    │         │
                                    │    FFN: ternary matmul
                                    │    128 → G dims, quantize
                                    │         │
                                    │    GSH: bucket, probe, k-NN
                                    │         │
                                    │    GSH pred
                                    │         │
                                    └────┬────┘
                                         │
                                    combination
```

## Execution order (critical)

**Step 1: direct quantization LSH (build first).**

Integrate `glyph_sig_quantize` into a new consumer tool. Use
direct ternary quantization of normalized pixels (and optionally
+ gradients) as the LSH signatures. Verify the new baseline
matches or exceeds the brute-force measurements (50.2% CIFAR,
88.9% Fashion).

**Step 2: FFN bridge (build second, on top of Step 1).**

Only after the direct-quantization LSH is verified.

## FFN specification

**Input:** 2M-dim routing vector (M=64):
- Dims 0..63: per-table 1-NN label, encoded as MTFP.
  label × (MTFP_SCALE / 9) so labels 0-9 span the MTFP range.
- Dims 64..127: per-table 1-NN distance, centered at median.
  (median - dist) × MTFP_SCALE / median.

**Transform:** one ternary matmul.
- Weight matrix W: 128 × G, ternary {-1, 0, +1}.
- G = 256 (2× expansion from input).
- Random weights initially (appropriate for non-spatial input).
- Output: G MTFP values.

**Nonlinearity:** ternary quantization.
- Compute tau from the training FFN outputs at the target
  density (same glyph_sig_quantize_tau method).
- Quantize each of the G output values to {-1, 0, +1}.
- Pack into G-trit signature (64 bytes at G=256).

**Output:** G-trit packed signature → GSH.

## GSH on FFN output

Standard Trit Lattice LSH on the G-trit FFN signatures:
- Bucket index keyed on first 16 trits (4 bytes)
- Multi-probe at radius 0, 1, 2 on the first 16 trits
- k-NN resolve by Hamming distance on the full G trits

The GSH finds training images whose FFN representations
(cross-table routing patterns) are nearest to the query's.

## Combination

Same as current layered_lsh: report agreement rate,
P(correct | agree), P(correct | disagree) for LSH vs GSH.
The FFN-GSH should produce a DIFFERENT error set than the
LSH — cross-table patterns detect confusions that per-pixel
distance misses.

## Weight generation

The FFN weight matrix W (128 × 256) is generated by the
standard RNG (glyph_rng). Each weight is drawn from
{-1, 0, +1} with equal probability. At density ~0.33, each
hidden unit reads ~43 of the 128 input dimensions and ignores
~85. The structural zeros determine which table signals each
hidden unit attends to.

## Implementation

In the tool (after Step 1's direct-quantization consumer):

```c
/* FFN: ternary matmul on routing vector. */
int FFN_IN = 2 * L1_M;       /* 128 */
int FFN_OUT = 256;
int ffn_sb = M4T_TRIT_PACKED_BYTES(FFN_OUT);

/* Generate FFN weight matrix (ternary, packed). */
glyph_rng_t ffn_rng;
glyph_rng_seed(&ffn_rng, ...);
/* ... generate and pack FFN_IN × FFN_OUT ternary weights ... */

/* Per query: */
m4t_mtfp_t routing_vec[128];
/* Fill routing_vec[0..63] = encoded labels */
/* Fill routing_vec[64..127] = encoded distances */

m4t_mtfp_t ffn_out[256];
m4t_mtfp_ternary_matmul_bt(ffn_out, routing_vec,
    ffn_weights_packed, 1, FFN_IN, FFN_OUT);

uint8_t ffn_sig[64];
glyph_sig_quantize(ffn_out, FFN_OUT, ffn_tau, ffn_sig);
/* → probe GSH bucket index with ffn_sig */
```

## Go / no-go for the FFN

**Prerequisite:** Step 1 (direct quantization LSH) must be
verified first. The FFN tests whether cross-table mixing adds
discriminative power ON TOP of a good LSH.

**Go:** GSH accuracy with FFN > GSH accuracy without FFN
(current multi-trit vote encoding) by ≥2pp. The FFN adds
genuine cross-table information.

**Strong go:** combined (LSH + FFN-GSH) > LSH alone by ≥1pp.
The FFN-GSH provides information the LSH cannot.

**No-go:** FFN-GSH ≤ current GSH accuracy. Cross-table mixing
via random ternary weights doesn't help. Would need
routing-learned weights or a different mixing structure.

## What the LMM found

The structural zero serves TWO jobs in the full architecture:
pixel attention in the direct quantization (which pixels matter)
and table attention in the FFN (which routing measurements
matter). Same ternary primitive, same mechanism, different input
domain. The architecture is self-similar — base-3 selective
attention at every layer.

Ternary matmul is appropriate for routing signals (low-dim,
non-spatial, expansion not compression) even though it's
deprecated for pixel signals (high-dim, spatial, compression).
The anti-random-projection rule applies to SPATIAL inputs, not
to ROUTING inputs.

The FFN must be built AFTER direct quantization LSH, not before.
The routing measurements the FFN reads must come from the
correct representation, not from the deprecated random-projection
path.
